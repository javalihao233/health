需求分析：
    用户在体检之前需要进行预约，可以通过电话方式进行预约，此时会由体检中心客服人员通过后台系统录入预约信息。
    用户也可以通过手机端自助预约。

预约流程如下：
    1、访问移动端首页
    2、点击体检预约【首页有多个选项】进入体检套餐列表页面
    3、在体检套餐列表页面点击一个具体套餐进入套餐详情页面
    4、在套餐详情页面点击立即预约进入预约页面
    5、在预约页面录入体检人相关信息点击提交预约
    6、预约成功后，跳转预约成功页面，显示预约前的注意事项，和相关预约信息。


    体检预约：会涉及到   检查项表   检查组表    套餐表   【检查组和检查项和套餐 是多对多对多关系】    预约【订单】表         6张表   4张主表  2张中间表

    一个套餐包含多个检查组
    一个检查组包含多个检查项
    一个订单对应一个套餐

    整个大体流程：

        [前台用户]
        1. 用户在首页点击体检预约按钮，会跳转套餐列表页面，每个套餐都会有一张套餐图。
        2. 用户选择好他需要的体检套餐，会跳转到套餐详情页面，在这个页面能看到该套餐的相关信息【涉及到3张表信息的展示】。
        3. 用户觉得是他需要的体检内容，觉得还不错。他会点击页面下方的一个确认按钮，跳转至咱们的一个订单信息页面。
        4. 用户填写他的一个基本信息【手机号码，验证码，体检日期等】提交。如果成功预约，会跳转到一个成功的页面，包含一些此次预约的基本信息以及一些体检注意事项。

        [后台用户]
        1. 首先我们得医生会录入医院所涵盖的体检信息，包括：检查项，检查组，套餐，预约设置。
           检查项和检查组和套餐 是多对多关系，都是一些CRUD

        2. 关于预约设置：
           我们前端封装了一个日历控件，可以非常方便直观的对具体某个日期做预约设置。
           另外为了方便医生的操作，我们还提供了批量预约设置， 【为啥要用POI呢 还没想好最佳的说法】

        3. 我们会在该页面提供一个固定模板的Excel文档，医生只需要对这个Excel简单进行操作即可。提高了用户的工作效率。
           【关于POI的一些操作  需要好生看看】

        4. 预约设置是与用户发起体检相关的，若该日子预约人数已满或该日子医生并没有进行预约设置，用户是不能对那天发起预约体检的。
